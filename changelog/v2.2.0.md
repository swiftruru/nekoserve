# NekoServe v2.2.0 -- Citations page hero: four interactive diagrams behind every stat pill

Released: 2026-04-24

v2.2.0 turns the four static "X papers / Y parameters / Z benchmarks" pills at the top of the Citations page into four **clickable tabs**, each backed by its own interactive visualisation. The number is no longer a claim the reader has to trust — clicking it reveals the actual papers, parameters, or benchmark data behind it, all in the same hero slot.

## The driving problem

Before v2.2, the Citations page lead with a hero title saying "my simulator is held up by 8 papers", followed by a single landscape diagram of those 8 papers. Next to the title sat three decorative stat pills: **13 papers**, **15 cited parameters**, **1 validation benchmark**. Two problems:

1. The title said 8, the pill said 13 — readers would reasonably ask which is right. The 13 counted the full bibliography (8 narrative papers + 5 methodology papers); the 8 counted only the narrative. There was no way to see the two groups separately.
2. Three of the four numbers pointed nowhere. There was no way to see which 15 parameters were cited, or which benchmark was being validated against.

v2.2 turns all four numbers into entry points. Each one now expands its own fully-annotated diagram in the same place the old landscape lived, so the hero section becomes a four-way interactive table-of-contents for everything downstream.

## The four tabs

All four tabs render inside a single `role="tabpanel"` container; clicking any pill swaps the diagram without the page scrolling. Keyboard navigation and ARIA labelling follow the same pattern the app's main nav already uses.

### Tab 1 — 故事主線 8 篇 (story narrative, default)

The existing three-tier landscape: Little (1961), Ancker & Gafarian (1963) ×2, Hasugian (2020), Dbeis & Al-Sahili (2024), Li (2025), Hirsch (2025), Ropski (2023). `CitationLandscapeMap` refactored to take a `variant` prop so story and methodology can share the same SVG layout while rendering different data.

### Tab 2 — 方法論/統計基礎 5 篇 (methodology landscape)

Brand-new landscape for the papers that back the **how** of NekoServe's validation: Sargent (2013), Kleijnen (1995) on V&V framework; Rubner (2000) on Earth Mover's Distance; Agresti (2013) on categorical data analysis; Wilson (1927) on confidence intervals. Same three-tier shape as tab 1 (indigo → amber → pink) so the visual grammar stays consistent; re-uses the landscape popover so clicking a node shows kind + contribution + DOI.

Each methodology paper also gets a full `story.landscape.details.<key>.kind / contribution` entry in both zh-TW and en locales, explaining in first-person why NekoServe uses it.

### Tab 3 — 15 個被引用的預設值 (parameter radial tree)

New component: `CitationParameterRadial`. Tree layout with **NekoServe capstone → 4 source-paper hubs → 15 parameter chips** arranged in vertical columns under each hub:

- **Hirsch 2025** hub — 8 params (`seatCount`, `staffCount`, `catCount`, `catInteractionTime`, `catIdleInterval`, `catRestProbability`, `catRestDuration`, `simulationDuration`)
- **Dbeis & Al-Sahili 2024** hub — 4 params (`customerArrivalInterval`, `orderTime`, `preparationTime`, `diningTime`)
- **Little 1961** hub — 2 params (`randomSeed`, `warmUpDuration`)
- **Ancker 1963** hub — 1 param (`maxWaitTime`)

Columns are sorted by param count (largest near centre). Every param chip shows the parameter name plus its literal default value + unit (`seatCount · 10 seats`, `catRestProbability · 0.2 probability`, etc.), extracted live from `PARAMETER_META` rather than hardcoded.

**Two independent popover paths:**

- Click a **parameter chip** → shows that parameter's default value, the rationale note from `PARAMETER_META`, and a link to the source paper's DOI.
- Click a **source-paper hub** → shows the APA short name, full title, DOI, the kind/contribution text (re-used from `story.landscape.details`), and a **list of every parameter that hub backs** as monospace pill badges.

Hover logic highlights the whole subtree: hovering a hub dims every chip that isn't below it; hovering a chip fades every hub that isn't its source.

### Tab 4 — 1 個驗證基準 (benchmark bar chart)

New component: `CitationBenchmarkBars`. Horizontal bar chart of the 9-state Hirsch 2025 ethogram that the validator scores simulation output against:

- 9 rows, sorted by empirical proportion (RESTING 31.7% → OUT_OF_LOUNGE 31.6% → SOCIALIZING 12.8% → HIDDEN 10.7% → … → PLAYING 0.3%)
- Each bar animates from 0 to its target proportion on mount (`cubic-bezier(.22,.9,.31,1.02)` over 900ms)
- **Wilson 95% CI whisker** drawn on every bar (horizontal line + end ticks in dark amber), computed per category from the published counts
- Percentage label anchored past whichever extends further — bar tip or CI upper bound — so the number is never covered by the error bar
- X-axis grid at 0% / 10% / 20% / 30% with soft dashed guides
- Header shows "227 小時 / 27 隻貓 / 12,505 次觀察取樣" as the methodological context

**Interaction:**

- Hover any row → ring highlight + a live-updating strip below the chart showing exact p, n/N, and the 95% CI bounds in monospace.
- Click any row → the full `StateDetailCard` springs into view below the chart. The card auto-scrolls into view on click (`scrollIntoView({ block: 'center' })`) so first-time users don't miss the response, and it carries a 600ms `neko-benchmark-card-in` animation with a brief orange glow pulse. A small bouncing "↓ 說明" chevron appears next to the selected row's percentage to reinforce the spatial relationship.

The detail card has four sub-sections:

1. **Header** with the Chinese state name + machine key + the three numbers (p, n/N, CI) rendered as `<InlineMath>` for typographic consistency with the rest of the Citations page.
2. **這個狀態是什麼 / What this state is** — per-state ethological description (what exactly counts as RESTING vs ALERT vs HIDDEN, how Hirsch draws the line). Wrapped through `renderWithTerms` so glossary jargon (ethogram, Wilson CI, Poisson, Little's Law, Pareto frontier, etc.) auto-gets hover tooltips.
3. **NekoServe 怎麼用 / How NekoServe uses it** — per-state explanation of how that category feeds into the simulator (FSM weight, interaction gating, CatWelfareScore contribution, playback animation frames, …).
4. **背後的公式 / The formula behind it** — full Wilson score interval as a `<BlockMath>`, then the **`InteractiveFormula`** widget (the same clickable-symbol-chip teaching pattern from the Results page) with 6 chips: `p̂_Wilson` / `p̂` / `n` / `z` / `z²/2n` / `±`. Clicking any chip expands a card with its label, description, unit, and a concrete example plugging in Hirsch's actual numbers ("HIDDEN this row has p̂ = 10.7%, Wilson-corrected centre ≈ 10.8%, CI = [10.1%, 11.3%]"). The shrinkage term and half-width get their own chips so the reader can see *why* Wilson differs from the normal approximation, not just what it looks like.

Three methodology tie-in chips at the bottom (`EMD · Rubner 2000`, `Wilson CI · Wilson 1927`, `χ² · Agresti 2013`) jump the user back to tab 2 when clicked, closing the loop: the benchmark is validated with exactly the five methodology papers in the previous tab.

## How the hero wires together

`CitationsPage` holds a `landscapeTab: 'story' | 'methodology' | 'params' | 'benchmark'` state. The four pills are `<button role="tab">` elements inside a `role="tablist"` wrapper, with `aria-selected` and `aria-controls` pointing at the shared panel. Active tab uses deep orange fill + `ring-2` ring; inactive pills keep the existing light-orange background so the grouping reads as "four related entry points, currently showing #3" instead of "three chips + one button."

```tsx
const [landscapeTab, setLandscapeTab] = useState<'story' | 'methodology' | 'params' | 'benchmark'>('story')

<div role="tablist" aria-label={t('story.stats.tablistLabel')}>
  <LandscapeTab active={landscapeTab === 'story'}       onClick={...} text={storyCount + ' papers'} />
  <LandscapeTab active={landscapeTab === 'methodology'} onClick={...} text={methodologyCount + ' methodology papers'} />
  <LandscapeTab active={landscapeTab === 'params'}      onClick={...} text={citedParamCount + ' cited parameters'} />
  <LandscapeTab active={landscapeTab === 'benchmark'}   onClick={...} text={'1 validation benchmark'} />
</div>
<div role="tabpanel" aria-labelledby={`landscape-tab-${landscapeTab}`}>
  {(tab === 'story' || tab === 'methodology') && <CitationLandscapeMap variant={tab} />}
  {tab === 'params'    && <CitationParameterRadial />}
  {tab === 'benchmark' && <CitationBenchmarkBars onJumpToMethodology={() => setTab('methodology')} />}
</div>
```

The count in each pill is now computed live from the underlying data (`CITATIONS` / `PARAMETER_META` / role filtering) rather than hardcoded, so adding a new methodology paper or parameter automatically updates the number and the tab content in lockstep.

## i18n additions

Both zh-TW and en citations locales gained three new top-level blocks, plus extended `landscape.details`:

- `story.stats.tablistLabel` + `storyCitations` / `methodologyCitations`
- `story.landscape.ariaLabelMethodology` + three `tierMethodology*` tier labels
- `story.landscape.details.{sargent2013vv,kleijnen1995vv,rubner2000emd,agresti2013cda,wilson1927ci}.{kind,contribution}`
- `story.radial.*` (ariaLabel, hoverHint, paramCount, sourceLabel, paramsListLabel)
- `story.benchmark.*` (title, method, hoverHint, ci95, nudge, meaningLabel, nekoUseLabel, sourcePrefix, formulaLabel, formulaIntro, chipEMD/Wilson/Chi2/Source)
- `story.benchmark.state.<KEY>` — 9 Chinese/English short names
- `story.benchmark.description.<KEY>` — 9 ethological descriptions (what counts as this state)
- `story.benchmark.nekoUse.<KEY>` — 9 NekoServe-side mappings (how the sim uses this category)
- `story.benchmark.formulaParts.{pWilson,pHat,n,z,shrinkage,halfWidth}.{label,desc,unit,example}` — 6 chip explanations with concrete Hirsch-number examples

All prose strings run through `renderWithTerms`, reusing the existing glossary infrastructure so terms like *ethogram*, *Wilson CI*, *Pareto frontier*, *CatWelfareScore*, *goodness-of-fit* auto-gain hover tooltips without any per-string annotation.

## Why this matters

The hero of a citations page is where a thesis examiner or a curious peer first decides whether the rest of the page is worth reading. v2.2 makes sure they can verify every headline number on the spot, without scrolling: the 8 papers are real papers with DOIs and roles; the 5 methodology papers each back a specific piece of the validator; the 15 parameters each resolve to a default value and a literature justification; the 1 benchmark is a full 9-category distribution with Wilson confidence bounds. This is the same auditability-first spirit as v2.1's Validation page rebuild, now applied one step earlier in the reader's journey.
