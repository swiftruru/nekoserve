# NekoServe v0.2.0 — Full Bilingual i18n

Released: 2026-04-13

## 🌐 Internationalization

- **Full bilingual coverage** — pages, charts, event log, error messages, learning sidebar, and the native Electron menu / About dialog all localized in Traditional Chinese (`zh-TW`) and English (`en`). No visible string is hardcoded in a natural language anymore.
- **One-click language toggle** — a single `🌐 繁中 / 🌐 EN` button in the header flips the entire UI. Selection persists across relaunches via `localStorage`.
- **Smart initial locale detection** — resolved in order: (1) saved user preference, (2) Electron `app.getLocale()` captured in preload, (3) `en` fallback. Resources are bundled and init is synchronous, so there is no first-frame language flash.
- **Type-safe translation keys** — `react-i18next` module augmentation anchors every `t('...')` to the canonical `zh-TW` JSON schema, so typos fail `tsc --noEmit` at compile time.
- **Dynamic event descriptions** — the 15 customer-journey event templates are now rendered via `` t(`events:${eventType}`, { customerId, resourceId, ... }) ``, with the Python-emitted `description` kept only as a debug fallback. Event log search matches the memoized rendered text so results stay consistent across languages.
- **Native Electron menu + About dialog** — a small `src/main/i18n.ts` string table drives the application menu. When the user toggles language, the renderer notifies main via a `locale-changed` IPC message and the menu rebuilds immediately.
- **Bilingual learning sidebar** — the classroom-notes content (queueing theory, Little's Law, utilization, bottleneck identification, SimPy reference, …) is fully rewritten in both languages.

## 🧯 Errors & exports

- `SimulatorError` is now a structured `{ type: SimulatorErrorType, error: string }` pair. The `error` field is an **English developer diagnostic**; the renderer always localizes via `` t(`errors:${type}`) ``. New typed codes: `TIMEOUT`, `BINARY_NOT_FOUND`, `NO_EXEC_PERMISSION`, `PARSE_ERROR`, `UNKNOWN_ERROR`.
- CSV exports now use stable **English `snake_case` headers** for cross-locale Excel compatibility. The event-log `description` column follows the current UI language so a CSV matches the screen the user was looking at.

## 🛠 Build / CI fixes

- **Windows build fix** — `scripts/build-simulator.ps1` was failing on `windows-latest` because PowerShell 5.1 reads `.ps1` files with the system ANSI code page (Windows-1252 on en-US runners), which corrupted the file's em-dashes and broke the parser. Replaced with ASCII hyphens; the Windows build now passes.
- **Node.js 20 deprecation opt-in** — added `FORCE_JAVASCRIPT_ACTIONS_TO_NODE24=true` at the workflow level to silence the 2025-09-19 deprecation warnings ahead of the 2026-06-02 forced upgrade.

## 📚 Documentation

- README now includes a full **Internationalization** section covering the resource layout, naming conventions, how to add a key, how dynamic event templates work, how error codes flow, and how to add a third language.

## ⬆️ Upgrade notes

- Version bumped `0.1.0` → `0.2.0`.
- The shared `SimulatorError.type` union is wider — if you consume this contract externally, update your typings.
- `scripts/build-simulator.ps1` is now pure ASCII; keep it that way to avoid PowerShell 5.1 encoding landmines.
