# NekoServe v0.7.1 — Update UX polish

Released: 2026-04-16

v0.7.1 polishes the update checking experience introduced in v0.7.0 with six targeted UX improvements. No new features, no simulator changes, no breaking changes.

## Accessible modal

The update modal now has proper accessibility attributes (`role="dialog"`, `aria-modal="true"`, `aria-labelledby`) and keyboard support. Pressing **Escape** dismisses the modal (except during the checking state). The primary action button auto-focuses when the modal transitions to an actionable state.

## Duplicate check prevention

Rapid clicks on the "Check for Updates" button or menu item are now ignored while a check is already in progress. A ref-based guard in `useUpdateCheck` prevents overlapping API requests.

## About page button loading state

The "Check for Updates" button on the About page now shows a spinner and "Checking..." text while a check is in progress, with the button visually disabled. This gives immediate feedback that the action was received.

## Retry button on error

When an update check fails, the modal now shows a **Retry** button (primary) alongside a dismiss button. Users can retry directly without closing the modal and navigating back to the menu or About page.

## Release notes preview

The "update available" modal now includes a collapsible **"What's New"** section that displays the changelog body from the GitHub release. The section defaults to collapsed and shows up to 800 characters of the release notes in a scrollable area. If the release has no body text, the section is hidden.

## Silent auto-check retry

The startup auto-check (5 seconds after launch) now retries once after 30 seconds if the first attempt fails. This handles the common case where the network is not yet ready when the app launches. The retry is silent and only fires once.
