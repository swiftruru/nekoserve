# NekoServe v0.6.1 — "Defend the Setup" pass

Released: 2026-04-15

v0.6.1 turns the Simulation Settings page into something you can actually defend in front of a teacher. Every default value on the page now has a structured design rationale, a clear theoretical framing, and clickable literature references. A new sidebar card lays out, in the author's own voice, what a scenario assumption is, why this app uses them, and how to strengthen that position with sensitivity analysis and partial calibration. No simulator changes, no event schema changes, no breaking changes. v0.6.0 result JSON replays pixel-identically.

## 📘 Per-parameter design rationale

Every section on the Settings page (🏠 Café resources, 👥 Customer behavior, ☕ Service time, 🐱 Cat interaction, 🎲 Simulation parameters) now has a collapsible **"Design rationale"** `<details>` block under the section title. Expanding it shows a structured rationale for every parameter in that section, with five fields:

- **Meaning** — what this parameter represents in the model.
- **Why it's needed** — which system behavior it controls; what would be missing if it were removed.
- **Theoretical basis** — the queueing-theory / DES concept it maps onto, with an inline KaTeX formula where relevant (for example `T ~ Exp(λ), λ = 1/8` for `customerArrivalInterval`).
- **Default-value basis** — how the default number was estimated (common sense, analogy, literature range).
- **📚 References** — a bulleted list of clickable citations that support the distribution or modeling framework.

All 14 parameters are covered. The rationale content is localized (zh-TW canonical, en parallel) via new `parameters.*.rationale` keys in `settings.json`. Default collapsed so the page looks the same at first glance; one click per section opens the whole block.

## 📚 Clickable literature references

Every citation in the rationale block, and every entry in the new bibliography card, is a live link that opens in the default browser via Electron's `setWindowOpenHandler` (already wired in `src/main/index.ts`). The reference schema changed from `string[]` to `{ text, url }[]`:

```jsonc
"references": [
  { "text": "Kleinrock (1975), Queueing Systems Vol 1: Theory, Ch 3",
    "url": "https://scholar.google.com/scholar?q=Kleinrock+Queueing+Systems+..." },
  { "text": "Little (1961), A Proof for the Queuing Formula: L = λW, Operations Research 9(3)",
    "url": "https://doi.org/10.1287/opre.9.3.383" }
]
```

URL strategy:

- **High-confidence DOIs** for the INFORMS Operations Research papers (Little 1961, Larson 1987, L'Ecuyer 1999) and the SIAM Review of Knuth's TAOCP Vol 2 resolve via `doi.org` directly.
- **Law (2010)** Winter Simulation Conference tutorial ("Statistical Analysis of Simulation Output Data: The Practical State of the Art") points straight at `dl.acm.org/doi/10.5555/2433508.2433519`.
- **Books, proceedings, and older Springer journals** use Google Scholar search URLs so the click always lands on a functional search result page, with no risk of dead DOIs.

A new reusable `<Ref>` primitive in `src/renderer/src/data/learnContent/shared.tsx` styles every citation with a dotted amber underline, and the same dotted-underline style is used inside the `ParamRationale` component so the two surfaces look consistent. Cursor on hover becomes clickable, external arrow opens in the system browser, tooltip shows the destination.

## 🎓 "A note about these numbers" sidebar card

A new top-of-page disclaimer banner on the Settings page and a new sidebar card in the Learning panel lay out the methodology stance in the app author's own first-person voice — not an AI or textbook voice. The card opens with:

> 我做這個 app 的時候，沒有真的跑去一間貓咪咖啡廳蹲點計時。設定頁上看到的每個數字，不管是座位 10 張、到達間隔 8 分鐘，還是貓咪 4 分鐘找一次人，都是我憑常識、對小店的生活觀察，以及對貓的想像所訂出來的。它們屬於**情境假設**，不是實地量測的資料。

The card then works through:

- **Why the app still makes sense** as a scenario-assumption model (qualitative what-if vs quantitative forecasting).
- **The distribution vs value distinction** — "picking Exp for arrivals is backed by Kleinrock; picking 8 as the mean has no literature behind it and that's fine, they're answered separately".
- **How to defend a scenario assumption** in writing: declare it, state where each guess came from, run a sensitivity analysis, do partial on-site calibration, cite a plausible range from literature where one exists.
- **What this app is good for and not good for** — classroom demos, what-if exploration, SimPy practice vs real investment decisions, government reports, precision-bounded science.

A parallel English translation of the same card, in the same first-person voice, ships with the `en` locale.

## 📚 Bibliography card

A new `references` sidebar card sits below "A note about these numbers" and lists the full bibliography grouped by topic:

- Queueing-theory foundations (Kleinrock, Gross & Harris)
- Core theorems (Little)
- Simulation methodology (Law & Kelton, Banks et al., Ross, Law 2010)
- Agent-based modeling (Borshchev & Filippov, Macal & North)
- Waiting-line psychology / reneging (Maister, Larson, Haight)
- Random number generation (Knuth, L'Ecuyer)

Each entry is wrapped in the same `<Ref>` primitive, so every bibliography line is a clickable link to the paper or book. The card ends with a reminder that these references support the distribution and framework choices, not the specific numerical defaults (those remain scenario assumptions).

## 🧩 New component: `ParamRationale`

`src/renderer/src/components/ParamRationale.tsx` is a small reusable `<details>` block that takes a list of parameter keys and renders them with the structured rationale fields plus the reference list. It:

- Pulls all copy from i18n (`settings:parameters.*.rationale.*` and `settings:rationaleSection.*`).
- Renders the formula via the existing `InlineMath` component when present.
- Defensively type-guards the references array so a malformed JSON entry just drops that row rather than crashing the page.
- Uses the existing cream / orange color tokens, so the block sits naturally inside each section card.

`SettingsPage.tsx` imports it once and drops one `<ParamRationale>` into each of the five sections with the relevant parameter keys.

## 📘 Academic disclaimer banner

A new top-of-page card appears immediately under the scenario-preset buttons on the Settings page. It states in the author's first-person voice:

> 這頁上每個預設值都是我憑常識與對小店的觀察訂出來的情境假設，不是從哪家真實店家量測來的。各區塊的「設計依據」展開後可以看到這個數字是怎麼估的、背後用什麼分佈，以及分佈本身的文獻依據。想知道整個方法論的立場，側邊的「關於這些數字，先說在前頭」那張卡寫得更完整。

This is the first thing a teacher sees on the Settings page, so the stance is declared up front instead of buried in a tooltip.

## ✍️ App copy voice

All new app-facing copy in this release is written in the user's own first-person voice, not a textbook or AI voice. This is a deliberate project convention going forward: every piece of in-app methodology prose should sound like the author explaining their own design choices to a reader, not a neutral knowledge dump. Earlier drafts of the scenario-assumption card that read like a textbook chapter were rewritten after user feedback.

## 🛠 Minor corrections

- Fixed the year in one citation: Law's "Statistical Analysis of Simulation Output Data: The Practical State of the Art" is **2010** (WSC 2010, pp. 65–74), not 2007. Earlier drafts had the wrong year; it is now linked to the exact ACM Digital Library DOI (`10.5555/2433508.2433519`).
- Knuth TAOCP Vol 2 citation link updated to `https://doi.org/10.1137/1012065` (SIAM Review of the 3rd edition) in response to a manual correction.

## ⬆️ Upgrade notes

- Version bumped `0.6.0` → `0.6.1`.
- **No breaking changes**. No simulator changes, no event schema changes, no metric field changes, no reducer state changes. v0.6.0 result JSON replays identically.
- The reference schema in `settings.json` changed from `string[]` to `{ text, url }[]`. This is a content schema change inside i18n resources, not a public API. Any third-party locale file (there is none today) would need the same shape.
- Two new sidebar cards on the Settings page: `scenario-assumption` and `references`. Existing cards (DES intro, Poisson, Normal, SimPy Resource, Little's Law) are unchanged.
- One new React component (`ParamRationale.tsx`) and one new shared JSX primitive (`<Ref>` in `learnContent/shared.tsx`). Both are pure additive imports.
- `prefers-reduced-motion` still respected everywhere; no new animations in this release.
