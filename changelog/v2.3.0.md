# NekoServe v2.3.0 -- Hirsch 2025 fidelity audit: cat-cat interactions, attention modes, area benchmark, paper-question disclosures

Released: 2026-04-26

v2.3.0 closes the gap between what the Hirsch et al. (2025) paper reports and what NekoServe actually validates against. v2.0-2.2 had the 9-state ethogram and vertical-level distribution wired up, but the paper's customer-flow rate, cat-cat interaction rate, four-mode human-cat attention mix, and three-area distribution were either uncited or uncited-but-unmodelled. This release adds the simulator events, validation benchmarks, and provenance disclosures so every quantitative claim Hirsch makes is either reproduced or honestly flagged as future work.

## What got modelled (simulator changes)

**Cat-cat interaction events (Hirsch §3.3, 0.58 events/cat/hr, 53/47 affiliative/agonistic split).** Two new event types `CAT_CAT_AFFILIATIVE` / `CAT_CAT_AGONISTIC`. A SimPy process samples interactions at the aggregate rate `catCount × 0.58/60` per minute, picks an initiator and partner from same-area in-lounge non-hidden cats, classifies via `Bernoulli(0.53)`. Three new metrics: `catCatAffiliativeCount`, `catCatAgonisticCount`, `catCatInteractionRatePerHour`. 24-hour Hirsch-config sim now reproduces the paper's rate to within ±3% on a fixed seed.

**Three-area time accumulator (Hirsch Figure 3 left panel).** New `area_time` accumulator parallels the existing `state_time` and `level_time`. Emits `catAreaShare: { AREA_1, AREA_2, CAT_ROOM }` in metrics output, validated against the paper's 45.2% / 23.2% / 31.6% breakdown.

**Lounge occupancy cap (Hirsch Methods §2.1, max 14 customers).** New `maxLoungeOccupancy` config field; when > 0, seat capacity is floored to `min(seatCount, maxLoungeOccupancy)` and overflow runs through the existing `maxWaitTime` abandonment flow. The canonical `hirsch-paper` preset leaves it disabled because Hirsch's behavior averages span LOW/MID/HIGH occupancy days; pinning to 14 would lock the sim into LOW only and bias RESTING / OUT_OF_LOUNGE upward. Available as an opt-in for what-if studies.

**Weekend arrival multiplier (Hirsch Figure 2, weekend median 84.5/day vs weekday 34/day, ratio ≈ 2.5×).** New `weekendArrivalMultiplier` config field; when > 1.0, days 5-6 of a 7-day cycle get arrival rate scaled by the multiplier. Default 1.0 keeps single-day sims unchanged.

**Maintenance routine events (Hirsch Methods §2.1, feeding 4×/day, litter cleaning 2×/day).** Two new event types `STAFF_FEEDING` / `STAFF_LITTER_CLEANING` fire at fixed within-day times (08:00 / 11:00 / 14:00 / 17:00 for feeding; 09:00 / 17:00 for litter). Pure observability markers; do not currently alter cat behavior.

## What got benchmarked (validation changes)

**Three-area benchmark (`benchmarks.ts: catArea`).** Adds Hirsch Figure 3 left-panel breakdown: AREA_1 5,653/12,505 (45.2%), CAT_ROOM 3,948 (31.6%), AREA_2 2,904 (23.2%), each with Wilson 95% CI and source citation. Validator now produces issues for area mismatches >10pp with `suggest.area.*` advisory text. Matching i18n entries for `AREA_1` / `AREA_2` / `CAT_ROOM` glosses.

**Human-cat attention 4-mode benchmark (`benchmarks.ts: humanCatAttentionMix`).** Adds Hirsch Figure 6 attention crosstab: `NO_INTERACTION` 44.4%, `NON_CONTACT_ATTENTION` 29.0%, `CONTACT_ATTENTION` 23.2%, `NO_ATTENTION_FROM_HUMAN` 3.4%, n = 3,310 classifiable customer-scan dyads (Women 1911 + Men 620 + Girls 447 + Boys 332). Surfaced in the provenance card with a 🚧 marker because NekoServe does not yet emit per-dyad attention classifications -- the paper's distribution is shown for transparency and future validation.

**Practical-equivalence band on CI checks.** Hirsch's n = 12,505 makes Wilson half-widths only ~±0.8pp -- too tight for Monte Carlo equivalence. The provenance table now uses three-tier symbols: ✓ inside CI, ≈ outside CI but within 3pp practical band (`PRACTICAL_BAND`), ⚠ off by >3pp. Symbol legend rendered in the table footer.

**`staffCount` sanity warning.** Validator flags `staffCount` outside [2, 3] (Hirsch's weekday-2 / weekend-3 setup) as a soft warning rather than a hard issue.

## What got disclosed (provenance changes)

**📝 Paper questions card.** New section in `BenchmarkProvenance` listing three points where Hirsch 2025 leaves loose ends:

1. **Figure 6 df mismatch** -- body text says χ²(18) = 87.42, caption says χ²(16) = 107.07, paper does not reconcile.
2. **In-lounge n = 8,547 vs 8,553** -- caption gives 68.4% × 12,505 ≈ 8,553, but the three vertical-level counts sum to 8,547. NekoServe commits to 8,547 (directly reported sum); the 10-scan gap is treated as cumulative rounding.
3. **Stay > 12 weeks reverts to floor** -- Figure 4(a) shows the pattern but Discussion offers no explanation.

These are flagged as research-skill demonstration: NekoServe reads the paper carefully enough to spot gaps and surfaces them in-app.

**⚠ Non-Hirsch threshold callout in Methodology section.** New amber card lists the 5 numeric knobs in NekoServe's validator that are *not* from the paper: `abandonRate ∈ [0%, 25%]` (industry expectation), `noInteractionRate ∈ [30%, 60%]` (soft envelope), `KS / KL soft-cap = 0.3` (Sargent face-validity), `composite weights 40/30/30` (V&V blend), `significant gap = 6pp` (heuristic). Keeps citations honest -- readers know which numbers are paper-derived and which are NekoServe judgment calls.

**Figure 6 caveat directly above the attention table.** The df=18 vs df=16 inconsistency is also rendered as a ⚠ caption above the four-mode table, not just buried in the paper-questions card.

## What got polished (UI / event log)

**Event log resourceId fixes.** Customer-side events (`CUSTOMER_ORDER`, `ORDER_START_PREPARE`, `ORDER_READY`, `CUSTOMER_START_DINING`, `CUSTOMER_FINISH_DINING`, `CUSTOMER_LEAVE`) now carry the customer's seat label as `resourceId`, so the resource column populates instead of showing `-`. Pre-seat events (`CUSTOMER_ARRIVE`, `CUSTOMER_WAIT_SEAT`, `CUSTOMER_ABANDON`) render as italic "等候中 / queueing"; staff-only events (`STAFF_FEEDING`, `STAFF_LITTER_CLEANING`) render as "全店員工 / all staff". `ORDER_START_PREPARE` description copy fixed: was "店員 {{resourceId}}..." which read awkwardly when resourceId was the seat label, now plain "店員開始為顧客 #X 製作餐點".

**Hirsch-paper jargon glossary (+14 terms).** Added to `glossary.ts` with full bilingual `results.json` entries: scan sampling / 掃描取樣, all-occurrence sampling / 全發生取樣, focal sampling / 焦點取樣, BORIS, in-lounge, Figure 3 (Hirsch 2025), Figure 6 (Hirsch 2025), Mann-Whitney U, effect size r, facultative sociality / 兼性社會性, feigned sleep / 假寐, affiliative, agonistic, occupancy level / 占用率分級. The provenance card's table titles, source columns, caveat lines, and paper-questions section all now run through `renderWithTerms`, so every Hirsch-specific term carries a hover tooltip.

**Validation table column-header tooltip on `n`.** Dotted underline + title attribute explaining "n = scan count for the category". Footer legend reiterates this plus the symbol meaning.

**Benchmark `note` localization.** Source-note strings (`n estimated from proportion × total scans`, `n reported directly: ...`) are now keyed and translated to zh-TW so the provenance card no longer leaks English mid-sentence.
