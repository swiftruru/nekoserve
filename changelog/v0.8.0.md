# NekoServe v0.8.0 -- UX overhaul: dark mode, onboarding, and quality-of-life

Released: 2026-04-16

v0.8.0 is a UX-focused release that adds 10 new features to improve the day-to-day experience of using NekoServe, headlined by a warm dark mode and a guided onboarding tour.

## Dark mode

A cozy "bark" dark theme that matches the cafe vibe of the light theme. Toggle via the moon/sun button in the header. The palette uses warm browns (#1a140e to #f0e8dd) instead of cold grays. Every page, card, chart SVG, KaTeX formula, learning panel, and modal is fully adapted. First-time users default to light mode; the preference persists in localStorage.

## Toast notification system

Visual feedback for actions that previously happened silently: export success, scenario save/delete, disabled-tab clicks. Toasts appear in the bottom-right corner with auto-dismiss after 3 seconds. Three types: success (green), error (red), info (orange).

## Keyboard shortcut help

Press `Cmd+K` (macOS) or `Ctrl+K` (Windows/Linux) to open a modal listing all available shortcuts. Also accessible via the `⌘K` button in the header.

## Onboarding tour

A spotlight-style 4-step guided tour for first-time users: scenario presets, run button, playback tab, results tab. The tour auto-scrolls to each target element and positions a tooltip with smart viewport clamping. Press `?` or click the header button to replay anytime. Clicking the dark overlay advances to the next step as a safety escape.

## What's New modal

On first launch after an update, a modal shows the new version number and a brief message. Dismissed state is tracked per-version in localStorage.

## Fullscreen playback

Press `F` while on the Playback page (or click the expand button) to hide the header, navigation, and sidebar for an immersive view. Press `F` again to restore.

## Drag-and-drop scenario import

Drop a `.json` file onto the Settings page to import simulation parameters. Accepts both raw config objects and `{ config: ... }` wrappers. Schema validation with toast feedback on success or failure.

## Smart number inputs

Parameter inputs now allow clearing to empty while editing. On blur, if the field is empty or invalid, it reverts to the last valid value. External changes (preset switch, reset, drag-drop import) sync only when the field is not focused.

## Disabled tab guidance

Clicking a grayed-out tab (Playback, Results, Event Log) before running a simulation now shows a toast: "Go to Settings and run a simulation first."

## Accessibility improvements

- Navigation tabs: `role="tablist"`, `role="tab"`, `aria-selected`
- Toast container: `role="status"`, `aria-live="polite"`
- Progress bar: `role="progressbar"` with `aria-valuenow`/`min`/`max`
- Global `focus-visible` outline for keyboard navigation
