# NekoServe v0.9.0 -- Accessibility: WCAG 2.1 AA compliance pass

Released: 2026-04-16

v0.9.0 is an accessibility-focused release that brings NekoServe closer to WCAG 2.1 AA compliance. Focus management, screen reader support, keyboard navigation, and semantic HTML are all significantly improved.

## Focus trap for all modals

A new zero-dependency `useFocusTrap` hook ensures Tab/Shift+Tab cycles within the active modal and focus is restored to the trigger element on close. Applied to all 5 modal/overlay components: UpdateModal, WhatsNewModal, ShortcutHelp, OnboardingOverlay, and InspectPopover. All modals now carry proper `role="dialog"`, `aria-modal="true"`, and `aria-labelledby`.

## Screen reader announcements

- **RouteAnnouncer**: a visually-hidden `aria-live="polite"` region announces the current page name whenever the user switches tabs (e.g. "Navigated to Settings").
- **Language switch toast**: changing locale fires a toast that is announced to screen readers via the existing `aria-live` toast container.
- **Filter result count**: the EventLogTable announces the number of matching events when search or type filters change.

## Skip to main content

A visually-hidden skip link appears on first Tab press, allowing keyboard users to jump past the header and navigation directly to the main content area.

## Icon-only button labels

Theme toggle, shortcut help, onboarding tour, and all modal close buttons now have `aria-label` so screen readers can identify them.

## Form validation accessibility

ParamInput gains a new `error` prop that wires up `aria-invalid` and `aria-describedby` for proper error announcement. The HelpButton tooltip now reports `aria-expanded` state.

## EventLogTable semantics

- Column headers have `scope="col"`
- Search input has `aria-label`
- Filter toggle buttons report `aria-pressed`
- Clickable rows are keyboard-accessible with `tabIndex={0}` and Enter/Space activation

## CafeScene keyboard access

Each seat and cat in the SVG playback scene is now keyboard-focusable (`tabIndex={0}`) with `role="button"` and a descriptive `aria-label` (e.g. "Seat 3, occupied"). Press Enter or Space to open the InspectPopover. The popover now also closes on Escape.

## Chart text alternatives

All 10 chart and visualization components have `role="figure"` and `aria-label` on their outer wrappers. The three Recharts-based charts (UtilizationChart, WaitTimeChart, CustomerPieChart) include visually-hidden text summaries of their data.

## Fullscreen button label

The playback fullscreen toggle now shows visible text ("Fullscreen" / "Exit fullscreen") instead of a bare symbol that was invisible under the custom cursor tooltip system.

## LearningPanel landmark

The `<aside>` sidebar now has `aria-label` for landmark identification by assistive technology.

## Progress bar label

The simulation progress bar (`role="progressbar"`) now has an `aria-label` describing its purpose.

## i18n

All new accessibility strings ship in both zh-TW and en: `common:a11y.*`, `eventLog:searchLabel`, `eventLog:filterResultAnnounce`, `playback:a11y.*`, `playback:fullscreen`, `playback:exitFullscreen`.
