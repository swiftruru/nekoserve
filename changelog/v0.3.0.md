# NekoServe v0.3.0 — Simulation Playback

Released: 2026-04-13

The big addition in this release is a fully animated replay of the simulation on a hand-drawn café floor plan. Run a scenario and NekoServe now lands you on the Playback page with the animation already playing; watch customers walk in, queue, get seated, order, dine, play with a cat and leave in real time, then click **View Results →** whenever you want the numbers.

## 🎞️ Simulation Playback page

- **SVG café floor plan** with seven zones: door, seat queue, N-cell seat grid, kitchen (M staff dots that light up when busy), cat queue, K-cell cat zone, exit. Cream-and-orange palette matches the rest of the app; hand-drawn zone cards with rounded corners and a subtle drop shadow.
- **Emoji customer avatars** on stage-colored chips, moving between zones with a 320 ms GPU-composited CSS transition. Stage glyphs: `🙂` waiting, `📝` ordering, `⏳` waiting for food, `🍽️` dining, `😺` playing with a cat, `😿` abandoning, `👋` leaving.
- **Auto-play on run** — finishing a simulation lands the user on Playback and starts playback from `t=0` automatically. A header button **📊 查看統計結果 / View Results** jumps to the statistics page for users who only want the numbers.
- **Playback transport**: play / pause, reset, five speed presets (0.5× / 1× / 2× / 4× / 8×, default 4×), step to previous / next distinct event (`⏮` `⏭`).
- **Draggable timeline scrubber** with a 60-bin event-density heatmap strip underneath, so students can spot the busiest moments at a glance and seek straight to them.
- **Speech bubbles** at key moments: `來囉 ✨` / `點餐了！` / `好好吃 🍽️` / `摸摸 💕` / `不等了 😤`. Stored in reducer state (not a side effect) so scrubbing the timeline deterministically rebuilds whichever bubbles should be visible at that sim-minute.
- **Click to inspect** any seat or any cat to pop up an `InspectPopover` card showing the current occupant's customer ID, lifecycle stage, and elapsed stay time. Click the same target again (or the background) to dismiss.
- **Keyboard shortcuts**: `Space` play / pause, `← →` seek ±10 minutes, `,` `.` step events, `0` reset, `1`–`5` change speed. A document-level `useKeyboardShortcuts` hook guards against firing while any `<input>` / `<textarea>` / `contenteditable` element has focus, so typing `Space` in the Event Log search box does not pause playback.
- **Per-frame performance**: the replay reducer is pure and O(N) on the ~600-800 event log; single-digit milliseconds per full replay, no snapshot index needed. Customer DOM nodes animate via `transform: translate3d` on SVG `<g>`, so the scene stays at 60 fps even with a full café.

## 🔗 Event Log ↔ Playback bidirectional linking

- `playbackTime` is now owned by `App.tsx` and shared by Playback and Event Log. While Playback is playing, the corresponding row in `EventLogTable` is highlighted (`bg-orange-100 ring-2`) and auto-scrolled into view (debounced 150 ms so rAF-driven updates don't thrash the scroll container).
- Clicking any row in the Event Log jumps straight to Playback with `simTime` seeded to that row's exact timestamp. The two pages are now two views of the same timeline.
- Running a new scenario resets the shared `playbackTime` to zero so replay always starts at `t=0`.

## 🧠 Architecture notes

- **Critical bug averted**: the Python simulator emits `座位-{seats.count}` as a seat resourceId, but that is the *count of currently-occupied seats at emit time*, not a stable seat slot identity — two customers can end up tagged `座位-1` at the same time. The new reducer ignores that label entirely and maintains its own free-pool virtual seat / cat slot allocators. Allocators are deterministic under re-replay.
- `usePlaybackClock` is a pure side-effect (no internal state): it runs a single `requestAnimationFrame` loop that advances the parent's `simTime` via refs on every frame while `playing` is true. This is what lets Playback and Event Log share a single source of truth without prop drilling.
- `applyEvent` is a single switch on 15 `EventType` cases composed out of four small allocators (`allocateSeat`, `releaseSeat`, `allocateCat`, `releaseCat`). A `ReplayContext` precomputes `catRestsByCustomer` so `CUSTOMER_FINISH_CAT_INTERACTION` can decide in O(1) whether to free the cat slot immediately or hold it for the upcoming `CAT_START_REST`.

## 🌐 i18n

- New `playback` namespace with full zh-TW and en coverage: title, subtitle, empty state, time labels, event progress, controls, speed, step labels, shortcut cheat sheet, zone labels, stage names, bubble captions, inspect popover labels. Registered in `src/renderer/src/i18n/index.ts` and typed via the `react-i18next` module augmentation so typos fail `tsc --noEmit`.
- Nav label `nav.playback` added to both locales.

## ⬆️ Upgrade notes

- Version bumped `0.2.0` → `0.3.0`.
- The `Page` union in `src/renderer/src/types/index.ts` now includes `'playback'`; `PAGE_ORDER` was extended between `eventlog` and `howitworks`. External consumers of `Page` should update their exhaustive switches.
- `usePlaybackClock` was rewritten as a controlled side-effect hook — `simTime` and `setSimTime` now come from the parent instead of being owned internally.
- No Python simulator, Electron main process, or CI workflow changes.
