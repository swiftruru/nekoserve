# NekoServe v1.1.0 -- Robustness, advanced exports, simulator upgrades, and UX polish

Released: 2026-04-17

v1.1.0 is a broad improvement release covering robustness, advanced analysis features, simulator upgrades, and numerous UX refinements across the entire app.

## Robustness & UX polish

- **Playback speed memory**: last-used playback speed persists across sessions via localStorage
- **Dynamic window title**: title bar updates to reflect the current page and simulation status (e.g. "NekoServe - Running...")
- **React ErrorBoundary**: page crashes now show a friendly fallback UI with a retry button instead of a white screen
- **Batch/sweep completion toast**: displays replication count and elapsed time on finish
- **Themed confirm dialogs**: history clear and delete now use in-app styled dialogs instead of native `window.confirm`
- **Parameter pre-run validation**: warns via toast for extreme parameters (arrival interval > duration, very short runs, excessive resources)
- **Simulation cancel button**: batch and sweep runs can be cancelled mid-execution with a red cancel button
- **Custom scenario validation**: `loadCustomScenarios()` now validates schema at load time, silently skipping corrupt entries

## Export & import

- **Batch summary CSV export**: `exportBatchSummaryCSV()` with mean, stdDev, CI bounds per metric
- **Sweep data CSV export**: `exportSweepCSV()` with per-point CI data for all metrics
- **CSV parameter import**: upload a CSV file with parameter sets, preview in a table, and run all sequentially

## Simulator upgrades (Python)

- **Wait time percentiles**: P50, P95, P99 for both seat wait and order wait, displayed below KPI cards
- **Warm-up period**: new `warmUpDuration` parameter resets all accumulators after the warm-up phase to eliminate initialization bias

## Event Log enhancements

- **Column sorting**: click time, type, or customer headers to sort (asc/desc/default)
- **Time range filter**: from/to inputs to isolate a specific time window
- **Event statistics summary**: collapsible badge bar showing event count by type
- **Customer journey tracking**: click a customer ID to filter all events for that customer with a lifecycle banner
- **Jump-to-time**: input field to scroll directly to a specific simulation minute

## Results page improvements

- **History panel pagination**: 5 entries per page with page number buttons (no more infinite scrolling)
- **Checkbox-based comparison**: select 2-4 runs to compare instead of comparing all history at once
- **Custom checkbox styling**: themed checkboxes matching the app design
- **Sticky comparison table header**: first column stays fixed during horizontal scroll
- **StayDistribution mean labels**: dashed mean lines now have visible "avg X.X" text annotations

## Content updates

- **About page**: updated description, 3 new usage steps (batch/sweep/challenges), new principle 6 (replication)
- **How It Works page**: 2 new sections (replication methodology, confidence intervals & sensitivity analysis)
- **Learning sidebar**: 5 new cards (batch replication, sensitivity analysis, confidence intervals, sweep chart reading, What-If Explorer)
- **Removed dead `version` key** from about.json (version is fetched dynamically)

## Themed UI components

- **ThemedSelect**: shared custom dropdown component replacing all native `<select>` elements
- **ConfirmDialog**: shared themed confirmation dialog extracted from ChallengeDrawer
- **ErrorBoundary**: new class component for graceful crash recovery
- **SVG icons**: dark/light mode toggle uses consistent SVG line icons instead of mixed emoji
