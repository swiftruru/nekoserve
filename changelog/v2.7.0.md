# NekoServe v2.7.0 -- Cumulative distribution (CDF) and attainment-probability analysis

Released: 2026-06-12

v2.7.0 adds the feature the mid-term committee asked for by name: take a metric's distribution and turn it into a decision screen that answers "what is the probability of clearing the bar?". A new CDF view joins the Live Mode metric detail page so the histogram (how often), the KDE density (what shape), and the CDF (how much probability has accumulated) all share one draggable threshold line. Drag the bar, read the attainment probability with a confidence interval, see the uncertainty band around the whole curve, or ask the inverse question ("how loose can my standard be if I want 75% to pass?"). Every chart and the readout carry an always-visible plain-language explanation, so a reader with no statistics background can follow along.

This release builds on the earlier (untagged) multi-domain generalization work; the CDF view works for any of the 20 tracked metrics, in any service domain.

## The new CDF view

**Three linked views, one threshold.** The Live Mode metric detail page now stacks the cumulative-mean chart, the histogram + KDE, and the new `CdfChart`, all sharing the same threshold line. Drag the bar on any of the three and the others plus every readout update live, because the line is a single value in `liveBatchStore` (the existing pass-bar system) and every panel reads off it. The histogram now also shades the passing side under the KDE curve, so the "area under the curve is the accumulated probability" idea is visible before you even get to the CDF.

**`CdfChart` (`components/live/CdfChart.tsx`).** Self-drawn SVG (no D3), matching the other live charts:

- **Empirical CDF** -- the exact staircase `F_N(x) = #{v_i <= x} / N`.
- **Smoothed CDF** -- the Gaussian-kernel curve `F̂(x) = (1/N) Σ Φ((x - v_i) / h)`, reusing the *same* Silverman bandwidth `h` the KDE density uses, so the smooth shape on the histogram and here tell one story. Toggle.
- **DKW 95% band** -- `F_N ± ε` with `ε = sqrt(ln(2/α) / (2N))`, a simultaneous confidence band over the whole curve that widens honestly when N is small. Toggle.
- **Threshold line** with a draggable grip, a guide line that reads `F(t)` off the y-axis, and a `1 - F(t)` label.
- **Hover** anywhere on the curve for `(x, F(x), 1 - F(x))`.
- **Suggested line** from the inverse lookup, drawn in a distinct blue dashed style.

**Attainment readout card (`components/live/CdfReadoutCard.tsx`).** Reads the attainment probability straight from the shared `calcExceedProb()` so it never disagrees with the histogram, then adds what the CDF analysis needs on top: the **Wilson 95% score interval** for that point estimate (stable near 0/1 and at small N), the curve readings `F(t)` and `1 - F(t)`, a PASS / FAIL verdict against a target attainment, and an auto-generated **bilingual conclusion sentence** with every number driven live by the calculation, never hardcoded. Example:

> 在目前設定下，貓咪福祉指數達到 3.50 以上的機率為 45%（95% 信賴區間 26% 到 66%），低於 50% 的目標，判定為未達標。
>
> Under the current setting, the probability that the cat welfare score reaches at least 3.50 is 45% (95% CI: 26% to 66%), below the 50% target. Result: fail.

**Inverse threshold lookup.** Enter a target attainment (default 50%) and the app works backwards to the standard that achieves it: higher-is-better uses `quantile(1 - p)`, lower-is-better uses `quantile(p)` (type-7 linear interpolation), then re-verifies against the empirical count so the suggested threshold genuinely clears the target. The suggestion is drawn as a separate line you can apply with one click.

**Plain-language explanations everywhere.** This is a teaching tool, so the "how to read it" is not hidden behind a toggle. The CDF chart carries an always-visible "🐾 這張圖在告訴我什麼 / What this chart is telling me" block explaining each line, the band, the threshold and `F(t)` / `1 - F(t)` in everyday words (and only mentions the smoothed / DKW / suggested lines when they are actually drawn). The readout card glosses all four numbers and the verdict in one plain line each.

**Persistence.** Each metric remembers its own target attainment and layer toggles (smoothed / DKW) in `localStorage` (`nekoserve:cdf-config`); the threshold and direction stay in the existing shared store.

## Calculation layer (pure, unit-tested)

A new framework-free module `utils/cdf.ts` holds all the math, with no UI dependencies: `empiricalCdf` (steps + a binary-search evaluator), `erf` / `normalCdf` (Abramowitz-Stegun 7.1.26), `smoothedCdf`, `wilson`, `dkwEpsilon` / `dkwBand`, `quantile` (type-7), and `thresholdForTargetP` (inverse lookup with re-verification). The attainment probability itself is not re-implemented; it reuses `calcExceedProb` from `utils/exceedance.ts`.

This is also the project's first unit-test suite: **Vitest** is added (`vitest.config.ts`, `npm test`), and `utils/cdf.test.ts` pins all of the above against the spec (empirical CDF monotonicity and ties, smoothed CDF bounds and bandwidth, Wilson at the extremes, DKW bracketing, quantile interpolation, inverse-lookup re-verification, and the higher/lower-better attainment cases). Vitest runs only the pure `src/renderer/**/*.test.ts` files and does not overlap with the Playwright E2E suite.

## Bug fix

**Live Mode empty-state crash (React #310).** The Live Mode page's empty-state early return sat *before* several hooks, so transitioning from the empty state to a running batch in place (for example via the empty-state quick-start button) changed the hook count and crashed with a minified React #310. The early return now lives after every hook. This was a pre-existing bug, surfaced while verifying the CDF view through that entry point.

## i18n

The `liveOverlay` namespace gains a full `cdf` block in both locales: chart title / axis / legend, the readout labels, PASS / FAIL, the inverse-lookup copy, the bilingual conclusion-sentence template (with `relGte` / `relLte` / `above` / `below` / `pass` / `fail` parts), and the always-visible `explain.*` and `readoutHelp.*` plain-language blocks. All bilingual, first-person voice, no em-dashes.
