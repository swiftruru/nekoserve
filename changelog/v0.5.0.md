# NekoServe v0.5.0 — "Live Classroom" Playback pass

Released: 2026-04-14

v0.5.0 turns the Playback page from a cute animation into an actual teaching tool. Characters now walk through real aisles, the shop gets atmosphere that tells you what time of day it is, and a brand-new **Live Learning Mode** overlay shows four core Discrete Event Simulation concepts ticking in real time next to the scene — with a Beginner / Pro toggle so both first-time students and professors can use the same window.

No simulator changes, no event schema changes, no breaking changes. Your v0.4.1 result JSON replays pixel-identically.

## 🚶 Real walking paths

Characters previously cut straight lines across the floor plan, ghosting through walls and other zone cards. Now every transition follows an L-shaped path that routes through the top or bottom aisle, so customers and cats visibly walk **around** the café instead of through it.

- **Shared geometry module** (`src/renderer/src/utils/cafeGeometry.ts`): zone constants, `seatPos`, `catHomePos`, `catVisitingPosForSeat`, and `staticCustomerPosition` moved out of `CafeScene.tsx` so both the render layer and the reducer agree on where every character belongs at the end of each stage.
- **Per-stage `stageStartPos` + `stageStartedAt` snapshot**: the reducer now records where and when each customer / cat entered its current stage. A dedicated `transitionCustomerStage` / `transitionCatState` helper centralizes all 14 state-changing event handlers so no handler can forget to snapshot.
- **Piecewise-linear path interpolator** (`pointOnPath`): six path functions (door → queue, queue → seat, seat → exit, queue → door for abandon, cat home → seat, seat → cat home) compute waypoints from the current `stageStartPos` and static target, and `pointOnPath` interpolates along arc length so characters move at uniform speed even when segments have very different lengths.
- **Ground-level door entries and exits**: the door-adjacent paths (arriving, leaving, abandoning) stay at door height (y=250) until they reach the queue column, then ride up — so new customers visibly walk in from the door instead of "falling from the ceiling". Customers leaving happy drop through a narrow corridor between the Cat zone and Exit card; abandoning customers route via the bottom aisle and slam out the front door.
- **CSS transform transition is disabled while walking**, per-frame JS positioning takes over. When stationary (queue shuffle, seat rearrangement) a short `transform 220ms ease-out` transition kicks back in for smoothness.

## 🎭 Dramatic event moments — CUSTOMER_ABANDON prototype

`CUSTOMER_ABANDON` used to be a quiet fade-out with a small "不等了 😤" bubble. It's now a full 1-sim-minute mini-skit:

- **Phase A (0.00–0.20 min) stomp in place**: the customer's body gets a `neko-drama-stomp` CSS class that bounces scaleY three times in ~240 ms. Walk progress is locked at 0 so they genuinely stay put while stomping.
- **Phase B (0.00–1.00 min) angry smoke trail**: three 💨 emoji rise staggered above the customer's head and follow them for the whole drama window via a decoration layer that reads `customerXY` every frame.
- **Phase C (0.20–1.00 min) slow angry walk**: abandoned walks now use a fixed 1.0 sim-min duration (vs the default ~0.5 sim-min for happy walks) so the stomp-out reads as deliberately slow.
- **Phase D (0.80–1.00 min) slam + dust cloud**: customer gets a `neko-drama-slam` class that scaleX-wobbles then shrinks to 0 sliding leftward, and a 💨 dust puff pops at the front door with a short `neko-drama-dust` keyframe.

The infrastructure is a new `'abandonDrama'` pulse kind with its own 1.0 sim-min TTL (per-kind TTL record now replaces the old global `PULSE_TTL_MIN` constant). Deterministic, scrub-safe, respects `prefers-reduced-motion`. This is the prototype for future per-event mini-skits.

## 🌸 Ambient scene decorations

The empty background now has time-of-day-aware atmosphere:

- **Drifting cherry blossom petals** (always on): 14 🌸 petals drift diagonally across the viewport with per-petal animation-duration / delay seeded from the petal index, so they stagger naturally across a ~14 s cycle.
- **Hanging yarn balls** (always on): 9 🧶 hang from the ceiling at y≈18 with a gentle 4.5 s sway, staggered 0.4 s per ball. Originally started as 🏮 lanterns but was replaced with yarn balls to fit the cat-café setting.
- **Morning butterflies** (`frac < 0.45`): 3 🦋 hover in a figure-8 path in the upper-left area during the first ~45% of sim time. Cross-fade to invisibility over a 0.1-frac window so there's a clean morning → midday handoff.
- **Dusk fireflies** (`frac > 0.6`): 7 ✨ twinkle in the upper area during the final ~40% of sim time with a yellow drop-shadow glow filter. Cross-fades in across the same 0.1-frac window.

All four layers are pure CSS keyframes so they cost nothing per React frame, and all are disabled by `prefers-reduced-motion`.

## 🎓 Live Learning Mode overlay

The headline feature. Playback now has a toggle-able **🎓 Live learning mode** panel that renders four live DES concept cards next to the scene, updating as the simulation plays. This is what makes NekoServe into an actual classroom tool instead of just a simulation demo.

### The four concept cards

1. **⏱️ Event-driven clock (C1)** — shows the current position on a mini event timeline with the nearest 9 events marked, a "next event in +X.XX min" countdown, and the running `state.appliedCount / eventLog.length` progress.
2. **📈 Queue length L(t) (C2)** — SVG polyline of queue length over the full run, with a dot marking the current time that color-codes by queue severity (blue / orange / red). Y-axis label shows the peak so scale is always obvious.
3. **⚖️ Little's Law (C3)** — three live numbers (L, λ, W) computed over a rolling 60-minute window, with the identity `L = λ × W` printed with actual values plugged in. Shows ✓ when within 20% tolerance, ≈ when not, and a "warming up…" hint during the first 15 sim-min to avoid transient spikes.
4. **🔥 Utilization ρ (C4)** — three bars for seat / staff / cat resources, color-coded (green / amber / red) with a 🔥 flag on whichever resource is currently the bottleneck.

### Two difficulty levels — Beginner / Pro

A pill switcher in the overlay header flips every card between two modes:

- **🐣 Beginner**: metaphor-driven views. C3 gets a water cup SVG (drops flowing in from above, water level = current L, drain at bottom); C2 shows a literal 👤 emoji queue; C4 labels each resource with a mood emoji ("quite free / getting busy / over capacity"); C1 highlights the "next thing happens" countdown with a friendly one-line hint. The per-card "Tell me more ▼" expand shows a **plain-language word formula** in a soft pill (e.g. "People in the cup = arrivals per minute × minutes each one stays") — no Greek letters, no KaTeX.
- **🎓 Pro** (default): the full professional view with L / λ / W / ρ symbols and KaTeX-rendered formulas in the expand (`L = λ·W`, `\bar{L} = (1/T) \int L(t) dt`, `\rho = busy / (c·T)`, `t_{next} = min\{...\}`). This is what a teacher wants to demo.

Same simulation, same state, two completely different presentations — flip the switch and the whole overlay updates instantly. No extra compute.

### Architecture

- **`buildSnapshotSeries(ctx, duration, stepMin=1)`** pre-computes metric snapshots every sim-minute when the result loads. 480 snapshots × ~500 events = ~240k reducer steps, runs well under a second on modern JS engines. Each concept card then does an O(log N) binary search (`snapshotAt`) per frame instead of re-running `replayUpTo`.
- **`avgOverWindow` / `deltaOverWindow`** helpers power the Little's Law rolling window without each card having to iterate the full series.
- Overlay lives in a new **`src/renderer/src/components/learning/`** directory: `LearningOverlay.tsx` shell + `ConceptCard.tsx` reusable shell + four `Concept*.tsx` components + `types.ts`.
- New `learnMode` i18n namespace (both locales) with full Beginner / Pro copy for every card.

### Side-by-side playback layout

When Learning Mode is on, PlaybackPage switches to a responsive grid: scene takes 60% of the row (`lg:grid-cols-[3fr_2fr]`), concept cards take 40% as a 2×2 grid that height-matches the scene. When off, the scene takes the full row again. Below `lg` (1024 px), the layout auto-stacks so mobile / narrow screens still work.

The toggle button lives in the Playback page header next to **📊 View results**, so it's always discoverable regardless of state — no more hunting at the bottom of the page.

## 🌐 Locale-neutral resource IDs

Previously the Python simulator emitted `resourceId = "貓-3"` / `"座位-1"` with Chinese characters baked in. These then got interpolated into English event-log descriptions as-is, producing "貓-3 walked over to customer #1" in the English UI. Fixed by:

- Python `core.py` now emits locale-neutral `cat-N` / `seat-N` resource IDs.
- New `events:resource.cat` / `events:resource.seat` i18n keys render them as "Cat #3" / "Seat #1" in English, "貓 #3" / "座位 #1" in Traditional Chinese.
- `EventLogTable` has a new `formatResourceId(raw)` helper used in both the Resource column and the description template `{{resourceId}}` interpolation. Still accepts the legacy `貓-N` / `座位-N` form for backward compatibility, so any old replay JSON still renders correctly.
- `parseCatSlotIdx` already accepted both forms, so the reducer didn't need any change.

## ⬆️ Upgrade notes

- Version bumped `0.4.1` → `0.5.0`.
- **No breaking changes**. No reducer state changes, no event schema changes, no config field changes. v0.4.1 result JSON replays identically (assuming it still used `貓-N` resourceIds — handled by the backward-compat branch in `formatResourceId`).
- `CafeState` interfaces gained optional `stageStartedAt` / `stageStartPos` (CustomerRuntime) and `stateStartedAt` / `stateStartPos` / `lastVisitSeatSlot` (CatSlot), all non-breaking additions used by the new walking-path and drama systems.
- `PULSE_TTL_MIN` changed from a single `number` constant to a `Record<PulseKind, number>` — any third-party code reading it needs to index by pulse kind. Internal only; not exposed in any public contract.
