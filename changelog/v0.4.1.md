# NekoServe v0.4.1 — "Clearly Cute" Playback polish

Released: 2026-04-14

This is a polish + reliability patch on top of v0.4.0. No simulator changes, no event schema changes — the café just feels more alive. If you liked the autonomous cat model but the scene still read as "shapes moving in straight lines", this release is for you.

## 🎨 Playback animation pass

Pure CSS keyframes, no new dependencies, respects `prefers-reduced-motion`, tempo auto-scales with replay speed so 8× never looks spastic.

### Character motion

- **Idle bob**: stationary customers and cats now gently bob up and down ±2 px like they're breathing.
- **Pop-in entrance**: freshly arrived customers scale in from the door with a springy overshoot (`cubic-bezier(0.34, 1.56, 0.64, 1)`).
- **Pop-out exit**: customers shrink and fade as they leave the scene.
- **Bouncier walk**: position transitions got a gentle overshoot curve so characters land with a micro-bounce instead of sliding into place.
- **Cat tail wag**: a visiting cat's sprite rotates ±3° back and forth while it's on a customer's lap.

### Character variety

- **Customer emoji pool** of 10 faces (🙂 😊 🤗 😃 🥰 🧑‍🎨 🧑‍💻 👧 👴 👵) picked deterministically by `customer.id % 10`. Same customer looks the same on every replay of the same seed.
- **Three cat species**: cat #1 is 🐱, cat #2 is 🐈, cat #3 is 🐈‍⬛, each with its own soft background tint (orange / yellow / purple). You can finally tell the cats apart at a glance.
- **Stage overlay badges**: instead of replacing the avatar emoji, stationary stages now keep the customer's personal face and overlay a tiny ⏰ (waiting), 📝 (ordering), or ⏳ (waiting for food) in the corner.

### Delightful details

- **Floating hearts**: 💕 and 💖 drift up from any customer with a cat on their lap.
- **Dining food bowl**: a small 🍜 pops up next to a customer while they're eating, then disappears.
- **Arrival sparkles**: the arrival speech bubble is surrounded by four ✨ that burst outward.
- **Kitchen steam**: three 〰️ rise from the kitchen zone while staff are busy.
- **Food + litter corners**: resting cats now drift between their home cell and either a 🍚 food corner or a 🚽 litter corner, deterministic by slot id + sim time. Zero simulation semantics — `catUtilization` still treats them as "resting". Just a story-ful way to animate the old `resting` state so students smile. (We explicitly decided **not** to model cat eating / toilet breaks as first-class DES phases because from the customer's queuing perspective they're indistinguishable from rest, and it would double the event schema without adding teaching value.)

### Zone feedback

- **Seat-full warn pulse**: when seat occupancy ≥ 80 %, the seat zone's border pulses red.
- **Door welcome flash**: the door zone briefly glows warm yellow when any customer arrives.
- **Exit flash**: the exit zone pulses green when a customer leaves happy, red when they abandon.
- **"All cats out" hint**: the cat zone shows a gray italic label when every cat is simultaneously visiting a customer, so students can explain why nothing is visibly at the cat zone.
- **Simulation ended banner**: when `simTime` reaches `simulationDuration`, a rounded banner appears at the top of the scene explaining that any customers still on-screen were mid-service when the clock stopped — it's not a bug, it's `env.run(until=...)` cutting off in-flight processes mid-way, and it matches the `arrived − served` difference students see in the KPI cards.

## ⚙️ Transport behaviour fixes

- **Reset now auto-plays**: clicking `🔄 重播 / Reset` jumps to t=0 **and** starts playback. Previously you had to press play again after reset, which was fiddly.
- **Scrub preserves playing state**: dragging the timeline no longer force-pauses. If you were watching, you keep watching from the new position when you release the slider. If you were paused, you stay paused.

## 🛠 CI reliability

The release workflow (tag push / `workflow_dispatch`) had two recurring reliability problems when GitHub's download CDN was flaky. Both fixed:

- **Linux AppImage is now actually built**. The release matrix previously only had `macos-latest` and `windows-latest`, so every tagged release since v0.1.0 advertised a Linux AppImage in README but never actually shipped one. `ubuntu-latest` is now in the matrix alongside `libfuse2` installation and `ci:pack:linux`. v0.4.0 was retroactively backfilled with its Linux AppImage via a `workflow_dispatch` re-run.
- **Transient 504s on electron + nsis downloads**. `npm ci` (which runs electron's postinstall) and the `electron-builder` packaging step can both hit `https://github.com/...` through a CDN that intermittently returns 504 Gateway Time-out for a few minutes. Both are now wrapped in `nick-fields/retry@v3` with 3 attempts and a 30-second backoff, so a single bad window no longer fails the whole matrix. Mirrored to `build.yml` for consistency.

## ⬆️ Upgrade notes

- Version bumped `0.4.0` → `0.4.1`.
- **No breaking changes**. This release is purely additive: no reducer state changes, no event schema changes, no config field changes, no metric field changes. Result JSON from v0.4.0 replays identically.
- `prefers-reduced-motion` now disables all of the new Playback animations (idle bob, hearts, steam, sparkles, tail wag, pop-in / pop-out, zone flashes). The scene stays functional — just without the movement.
