# NekoServe v1.2.1 -- Windows simulator fix and cursor stability

Released: 2026-04-21

v1.2.1 is a bug-fix release addressing two Windows-only issues reported by a classmate during testing.

## Windows simulator PARSE_ERROR

Simulation failed on Windows with `Failed to parse simulator stdout JSON` while running fine on macOS. Two root causes:

- **Main process (simulator-bridge.ts)**: stdout chunks were decoded individually as UTF-8 the moment they arrived. When a multi-byte character (any Chinese character in the event log) straddled a chunk boundary, the partial decode produced replacement characters and broke JSON parsing. Windows pipe buffers are smaller than macOS, so the output split more often and the bug manifested there. Now buffered as `Buffer[]` and decoded once on `close`.
- **Python simulator (`__main__.py`)**: `sys.stdout` defaulted to the Windows console codepage (cp950 / cp936), which can't encode some of the Chinese characters emitted in the result. Forced UTF-8 via `reconfigure(encoding="utf-8")` on stdio at startup.

## Cursor stability

The custom cursor replacement had recurring sync / visibility issues across platforms. Reverted to the native OS cursor for all normal use and kept only the hover cat: a small paw sprite now appears beside the native pointer when it hovers an interactive element, without replacing the OS cursor itself.

- Removed `cursor: none` rules and the full-screen overlay logic.
- `CustomCursor` component trimmed to a hover-only companion sprite with a (14, 14) offset from the pointer tip so it never covers the click point.

## Tooling

- Added `npm run screenshots` script and Playwright config for capturing slide-deck screenshots.
- `.gitignore` ignores auto-captured `docs/screenshots/` output.
