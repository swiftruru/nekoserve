# NekoServe v1.0.0 -- Classroom-ready: history, batch CI, sensitivity analysis, challenges, What-If, print & screenshot

Released: 2026-04-17

v1.0.0 is a major feature release that transforms NekoServe from a demonstration tool into a self-contained classroom platform. Eight new features span persistent history, DES methodology (multi-seed replication with confidence intervals), interactive parameter exploration, guided challenges, and export capabilities.

## Persistent simulation history

Simulation results are now stored in IndexedDB and survive app restarts. The Results page has a collapsible history panel where users can browse, load, rename, and delete past runs. No more losing results when the window closes.

## Multi-seed batch run with 95% confidence intervals

A new "Batch mode" toggle on the Settings page lets users run the same configuration N times (2-50) with different random seeds. KPI cards display mean +/- 95% CI using t-distribution critical values. A purple banner on the Results page indicates batch results. This addresses the most fundamental gap in DES methodology: distinguishing system behavior from random noise.

## Parameter sensitivity analysis (sweep)

A new "Sensitivity analysis" mode sweeps one parameter across a user-defined range. Results are displayed as an interactive line chart (SweepChart) with a metric selector dropdown. Each sweep point can optionally run multiple replications for CI bands. The chart uses Recharts LineChart with a confidence band overlay.

## Classroom challenges

Eight graduated challenges (3 easy, 3 medium, 2 hard) are built into a slide-out drawer accessible from the navigation bar. Each challenge has a goal condition, up to 3 progressive hints, and an explanation shown on completion. Progress is persisted in localStorage with a themed reset dialog. Challenges include: low abandon rate, high throughput, staff efficiency, cat paradise, balanced cafe, speed service, max throughput, and minimal resources.

## What-If Explorer

An interactive panel on the Results page lets users tweak 4 key parameters (seats, staff, cats, arrival interval) via sliders. Changes trigger a debounced re-simulation (500ms) and display a side-by-side comparison table with delta highlighting (green = improved, red = worsened). What-If results do not pollute the main history.

## Print / PDF export

A print button on the Results page triggers `window.print()` with a dedicated `@media print` stylesheet that hides navigation, sidebar, cursor overlay, and forces light mode. Charts expand to full width with page-break-avoid on cards.

## Playback screenshot export

A camera button on the Playback controls captures the current cafe scene via Electron's `webContents.capturePage()` and saves it as PNG through a native save dialog. The IPC handler is registered in the main process with a `capture-screenshot` channel.

## Scenario config diff in comparison view

The comparison table now shows a "Parameter differences" section at the top, highlighting only parameters that differ across the compared runs. Changed values are displayed with amber highlighting.

## UI polish

- Dark/light mode toggle icons replaced with consistent SVG line icons (Lucide style) instead of mixed emoji
- WhatsNew modal close button now has a proper square focus outline
- Fullscreen state automatically clears when navigating away from the Playback page (fixes a bug where header/nav could disappear on other pages)
- Batch and sweep mode toggles redesigned as selectable cards with descriptions instead of flat checkboxes
- Native `<select>` dropdown in sweep mode replaced with a themed custom dropdown matching the app's design language
- Challenge drawer shows an inline warning when no simulation results are available, instead of silently disabling interaction
